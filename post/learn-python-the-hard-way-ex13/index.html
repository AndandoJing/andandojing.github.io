Learn Python the Hard Way-ex13

· 2019-05-28 ·

from sys import argv script, first, second, third = argv print("The script is called:", script) print("Your first variable is:", first) print("Your second variable is:", second) print("Your third variable is:", third)

终端输入python ex13.py first second third 终端的python ex13.py命令种，ex13.py部分就是所谓的“参数（argument）” 这章说的就是写一个可以接受参数的脚本

import语句： 将python的功能引入脚本。但实际上术语应该称这些“功能”为模组（modules） argv，所谓的“参数变量（argument variable），是一个非常标准的编程术语。这个变量包含了你传递给python的参数 第3行将argv“解包（unpack）”，与其将所有参数放到同一个变量下面，我们将每个参数赋予一个变量名：script，first，second，third。解包的含义就是：把argv中的东西解包，将所有的参数一次赋予左边的变量名

如果给的参数不够，会出现错误 举例： File "ex/ex13.py", line 3, in script, first, second, third = argv ValueError: need more than 3 values to unpack

还有一种情况是 ValueError: too many values to unpack (expected 4) 这是因为实际在输入python 13_1.py时，后面跟的是除了script外的几个变量。举例来说： #从sys模组导入argv参数变量 from sys import argv #认定name, location, age, nationality为参数变量 script, name, location, age, nationality = argv print("Who are you?",) name = input() print("Where are you?",) location = input() print("How old are you?",) age = input() print("Where are you from?",) nationality = input() print("The script is called:", script) #这一行可以去除不显示 print("So, your name is:", name) print("And you are now in:", location) print("Now you are %r years old" % age) print("You are from:", nationality)

上面这样结合input()和argv，在终端执行时应该写入python ex13_1.py Jing Spain 28 China 而不是python ex13_1.py script Jing Spain 28 China

无论如何，在script, name, location, age, nationality = argv 这一行中，script这个占第一的词要在，后面可以不显示

argv和input( )有什么区别？ argv：参数是在用户执行命令时就要输入 input( ): 脚本运行过程中需要用户输入 命令行参数是字符串，就算在命令行输入数字，也需要用int()把它先转成数字
