这部分实际上是在终端输入python后，一行一行输入后可以得出结果。
所有的布尔逻辑表达式都可以用简单流程得到结果：

  1. 找到相等判断的部分（==或者!=），将其改写为最终值（True或者False）
  2. 找到括号里的and或者or，先算出他们的值
  3. 找到每一个not，算出其反过来的值
  4. 找到剩下的and或者or，解出他们的值
  5. 全部做完后，得出最后的结果是True还是False

举一个例子：
3 != 4 and not ("testing" != "test" or "Python" == "Python”)

解答过程：

  1. 拆分出3 != 4为True：True and not (“testing” != “test” or “Python” == “Python”)
  2. “testing” 1= “test” 为True：True and not (True or “Python” == “Python”)
  3. “Python” == “Python”: True and not (True or True)
  4. 找到括号中的and和or: (True or True) 为True：True and not (True)
  5. 找到not并将其逆转： not (True) 为False：True and False
  6. 找到剩下的and和or，解出值：True and False为False

⚠️注意：
虽然字节符中’testing’和”testing”是一样的，但是如果在python中直接输入：
’testing’ = “testing” => False
如果输入”test” and “test” => “test”
如果输入 1 and 1 => 1
python中返回两个被操作对象中的一个，会得到第一个操作字元。
如果输入False and 1 => False
如果输入1 and False => False
如果输入True and 1 + 0 => 1
如果输入1 + 0 and False => False
也就是说and和or运算符总会返回对象（不是左边的就是右边的）

短路逻辑：
任何False开头的and语句都会处理成False，并不会继续检查后面的语句。
任何包括了True和or的语句，只要处理到True字样，就不会继续向下推算，直接返回True。
不过尽量保证整个语句能正常处理。
