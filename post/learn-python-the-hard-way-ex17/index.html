上面是正常ex17.py的代码内容，如果用text.txt作为要复制的文件，copied.txt作为复制文件的目标，会显示：

Copying from test.txt to copied.txt
The input file is 16 bytes long
Does the output file exist? False
Ready, hit RETURN to continue, CTRL-C to abort.
>
Alright, all done.

在这个ex17.py里，import了新的模组exists，这个命令将文件名字符串作为参数，如果文件存在则返回True，否则返回False

这里我在最后多出了一个打印命令，让把复制后的新文件打印出来：

from sys import argv
from os.path import exists
script, from_file, to_file = argv
print("Copying from %s to %s" % (from_file, to_file))
#We could do these two on one line too, how?
in_file = open(from_file)
indata = in_file.read()
print("The input file is %d bytes long" % len(indata))
print("Does the output file exist? %r" % exists(to_file))
print("Ready, hit RETURN to continue, CTRL-C to abort.")
input(">")
out_file = open(to_file, 'w')
out_file.write(indata)
print("Alright, all done.")
out_file.close()
in_file.close()
new_file = open(to_file)
new_txt = new_file.read()
print(new_txt)

⚠️注意：如果按这种方式写，一定要把out_file.close()和in_file.close()写上，否则文件并没有被保存进磁盘，只是内存而已。

如果这里不加最后的打印步骤，可以在Linux或者OSX下面，在终端里直接输入cat copied.txt命令，会显示：

xiaojinyudeMBP:AtomProjects isaking$ cat copied.txt
last new line 1.xiaojinyudeMBP:AtomProjects isaking$

这个命令的用处是将两个文件“连接（concatenate）到一起，不过实际上它最大的用途是打印文件内容到屏幕上。
cat命令：
命令格式：
cat [选项][文件]…
命令功能：

  1. 一次显示整个文件：cat filename
  2. 从键盘创建一个文件：cat>filename 只能创建新文件，不能编辑已有文件
  3. 将几个文件合并为一个文件：cat file1 file2 > file

在ex17_1.py中对原本的脚本进行修改，尝试操作其他文件，看看是否能够正常使用(尝试将ex15_sample.txt复制到ex17_1_copied.txt)：

from sys import argv #导入argv模组
from os.path import exists #导入exists模组
script, old_file, new_file = argv #赋予参数
print("Copying from %s to %s." % (old_file, new_file))
#We could do these two on one line too, how?
data = open(old_file).read() #尝试用一行表达
print(data)
print("The old file is %d bytes long." % len(data)) #就文件多长？
print("Does the new file exist? %r" % exists(new_file)) #查看新文件是否存在
print("Ready, hit RETURN to continue, CTRL-C to abort.")
input("Your choice is:")
out_file = open(new_file, 'w')
out_file.write(data) #尝试用一行表达
print("Alright, all done.")
out_file.close()
open(old_file).close()

上面这样是可以正常运行的，脚本运行结束后在终端用cat ex17_1_copied.txt也可以正常显示出复制后的文本内容。

len() 函数的功能是：以数字形式返回传递的字符串的长度。
例如：
len(“abcde”)

read() 一旦运行，文件就会被读到结尾并且被close掉。

和sys一样，os也是python自带的基本库中的一员，主要提供和操作系统有关的功能。path提供了和路径操作有关的功能。

这里尝试把脚本精简，保存为ex17_2.py，将text.txt复制到ex17_2_copied.txt：

#这是精简ex17.py的尝试1
from sys import argv #载入sys.argv模组
script, from_file, to_file = argv #解包argv，将脚本名赋值给变量script，将参数赋值给from_file和to_file
#we could do these two on one line, how?
indata = open(from_file).read()#打开需要复制的文件并读取内容
open(to_file, 'w').write(indata)#打开需要复制到的文件并写入之前读取的就文件内容

结果：已经去除所有提示语，因此运行结束后不会打印任何东西，但终端cat ex17_2_copied.txt可以显示出已经成功复制。
再次尝试继续精简，保存为ex17_3.py，将ex12.html复制到ex17_3_copied.html：
