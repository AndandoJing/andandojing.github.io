close：关闭文件
read：读取文件内容，可以把结果赋给一个变量
readline：读取文本文件中的一行
truncate：清空文件，需要小心使用
write(stuff)：将stuff写入文件

from sys import argv #从sys导入argv模组
script, filename = argv #将argv解包，并将脚本名赋值给变量script；将参数赋值给变量filename

target.write(line3)

这个方法里new_text = input()，那么只能在终端输入一行内容，不能像之前line1=input()那样输入多行。在最后让把新内容读取打印出来了，比较方便看是否真的写入进去了。

因此可以再改一种输入方式：

from sys import argv
script, filename = argv
print("Now we're going to erase the %r." % filename)
print("If you don't want to erase it, hit CTRL-C (^C).")
print("If you want to erase it, hit RETURN.")
input(">")
print("We're opening the file...")
target = open(filename, 'w')
print("We're going to truncate the %r. Goodbye!" % filename)
target.truncate()
print("Now what do you want to write to the new %r?" % filename)
line1 = input("line1: ")
line2 = input("line2: ")
line3 = input("line3: ")
print("We're going to write these three lines to the %r." % filename)
target.write('\n'+line1+'\n'+line2+'\n'+line3+'\n')
print("Now we're going to close the file to save.")
target.close()
print("Ok, now here is your new file:")
file = open(filename)
print(file.read())

这里需要注意，如果写成
target.write(‘\n’, line1, ‘\n’, line2, ‘\n’, line3, ‘\n')
会报错
TypeError: write( ) takes exactly one argument (7 given)
也就是说write里只能输入一个string，所以要用+连接起来。

用格式化字符更改的话可以是：
target.write(‘%s\n%s\n%s\n’ % (line1, line2, line3))
或者用新format()格式串：
target.write(‘()\n()\n()\n’.format(line1, line2, line3))

🤔思考：代码里open(filename, ‘w’)这里的’w’是什么意思？

找到一些open的打开方式：
r:只读;
r+:读写，不创建.可读可写，若文件不存在，报错。r+会对之前写入的东西进行覆盖写（见例1）;
w:新建只写，不能读出;w和w+这两个都会将文件内容清零;
w+:新建写读，可读写.可读可写，若文件不存在，创建;
a:附加写的方式打开，不可读;
a+:附加读写的方式打开

w是写入模式：write
r是读取模式：read
a是追加模式：append
+是修饰符，文件将以同时读写的方式打开

例1:

fd = open(“1.txt”, ‘w+’)
fd.write(‘123’)
fd = open(“1.txt”, ‘r+’)
fd.write(‘456’)
fd = open(“1.txt”, ‘a+’)
fd.write(‘789’)

结果是456789

总结：
不可读的打开方式：w和a
若不存在会创建新文件的打开方式：a，a+，w，w+

open本身对于文件的写入操作态度是安全第一，所以只有特别制定以后，才会进行写入操作。
如果只写open(filename)会默认用’r’ 也就是read的模式打开，这是open()的默认工作方式
